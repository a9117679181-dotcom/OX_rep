# Prompt Plan

Промпты должны помогать человеку управлять Codex через ясные цели, ограничения и формат результата.

Стартовые типы промптов:
- превратить сырой бриф в структуру проекта;
- создать план сценария без полного текста;
- подготовить план демо;
- проверить материалы на завышенные обещания;
- составить список вопросов для человеческой проверки.

Каждый промпт должен явно указывать, что Codex готовит черновик, а человек проверяет и принимает решения.
